Your First Data Structure

1. What Is an Array?

An array is a collection of elements stored in contiguous memory locations.

In Python, we use lists, but conceptually they behave like arrays. 

numbers = [10, 20, 30, 40]
Key Characteristics
  • Ordered
  • Indexed (starts at 0)
  • Stores elements of the same conceptual type
  • Fast access by index

2. Why Arrays Exist

Arrays solve a simple problem:

"How do I store many values and access them efficiently?"

Without arrays, you'd need separate variables:

a = 10
b = 20
c = 30
d = 40

Arrays give you:

  • ‚Üí Structure
  • ‚Üí Predictability
  • ‚Üí Performance

3. Indexing: The Superpower of Arrays (O(1))

numbers = [10, 20, 30, 40]

print(numbers[0])  # 10
print(numbers[2])  # 30

No matter how big the array is:

  • Accessing numbers[i] takes the same time
üëâ O(1) index access

Mental Model

Imagine a row of mailboxes with numbers on them.
You don't open every mailbox ‚Äî you go directly to the one you want.

4. Traversing an Array (O(n))

for num in numbers:
    print(num)
4 elements ‚Üí 4 operations
n elements ‚Üí n operations
üëâ O(n) traversal

This should feel intuitive after Lesson 2.

5. Updating Values by Index (Still O(1))

numbers[1] = 99

Why?

  • Python already knows where index 1 lives
  • No shifting
  • No looping
üëâ O(1) update by index

6. Searching in an Unsorted Array (O(n))

numbers = [7, 14, 21, 28]
target = 21

for n in numbers:
    if n == target:
        print("Found")

Worst case:

  • Target is last
  • Or not present

You must check every element.

üëâ O(n) unsorted search
üëâ This is why algorithms matter, not just data structures.

7. Appending to an Array (Almost O(1))

numbers.append(50)

Most of the time:

  • Adds element to the end
  • Very fast

Occasionally:

  • Python needs more memory
  • Copies elements
üëâ O(1)* amortized
‚ö†Ô∏è

Don't worry about "amortized" yet

Just remember append is usually fast

8. Inserting in the Middle (This Is Expensive)

numbers.insert(1, 15)

What happens internally:

  • Elements after index 1 shift right
  • Many assignments occur

If array has n elements ‚Üí up to n shifts

üëâ O(n) middle insertion
üëâ This is one of the biggest weaknesses of arrays.

9. Removing Elements (Also O(n))

numbers.pop(1)

Same problem:

  • Elements must shift left
  • Cost grows with size
üëâ O(n) removal (not from end)

10. Summary Table (Very Important)

Operation Time Complexity
Access by index O(1)
Update by index O(1)
Traverse O(n)
Search (unsorted) O(n)
Append (end) O(1)*
Insert / delete middle O(n)

* Amortized

11. Tiny Exercises (Think, Don't Code)

1

Why is index access O(1) but search O(n)?

Index gives direct position (like a house address). Search must check each element until found.
2

If an array doubles in size, what happens to traversal time?

Traversal time also doubles ‚Äî O(n) means linear relationship with size.
3

Why might arrays be a bad choice for frequent insertions?

Each insertion requires shifting elements ‚Äî O(n) cost adds up quickly with frequent inserts.

12. Key Takeaway (Lock This In)

Arrays are fast for reading, slow for shifting.

If you remember only one sentence from this lesson, make it that.

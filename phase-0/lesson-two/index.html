1. What Is Time Complexity?

Time complexity is a way to describe how the runtime of an algorithm grows as the input size increases.

It's not about measuring actual seconds ‚Äî it's about understanding the relationship between input size and the number of operations.

Understanding the Notation: O(n)

When you see something like O(n), here's what each part means:

Symbol Meaning
O "Order of" ‚Äî a mathematical notation that describes growth rate
n The input size ‚Äî how many items you're working with

So O(n) reads as "Order of n" and means: the number of operations grows proportionally with the input size.

Common Misconception: O(1) ‚â† "One Operation"

O(1) does not mean "exactly one operation."

It means the number of operations stays constant ‚Äî it doesn't change as input grows. Whether it's 1 operation, 5 operations, or 100 operations, if that number doesn't depend on n, it's O(1). 

def example(arr):
    print(arr[0])      # 1 operation
    print(arr[0] * 2)  # another operation
    print(arr[0] + 10) # another operation
    # Still O(1) ‚Äî always 3 operations, regardless of array size

2. What Problem Does Time Complexity Solve?

Imagine you write a program and ask:

"Will this still work if my input becomes very large?"

Time complexity helps answer:

  • Will it be fast enough?
  • How does runtime grow as input grows?

3. The Core Idea (In One Sentence)

Time complexity describes how the number of operations grows with input size.

Input size is usually called n.

4. Let's Start with the Simplest Example

Example: One Loop
numbers = [2, 4, 6, 8, 10]

for num in numbers:
    print(num)

Let's analyze:

If list size = 5 ‚Üí loop runs 5 times
If list size = 100 ‚Üí loop runs 100 times
If list size = n ‚Üí loop runs n times
üëâ O(n) "order of n"

Meaning:

  • Runtime grows linearly
  • Double the input ‚Üí double the work

5. Why We Ignore Exact Numbers

for num in numbers:
    print(num)      # 1 operation
    print(num * 2)  # another operation

Runs: 2 operations per element = 2n total

We still say: O(n)

Constants (like 2, 5, 100) don't matter for growth

Growth trend matters

6. Constant Time: O(1)

numbers = [10, 20, 30]

print(numbers[0])

No matter:

  • List size = 3
  • List size = 1 million

This always takes the same time.

üëâ O(1) constant time

7. Two Loops = O(n¬≤)

numbers = [1, 2, 3]

for i in numbers:
    for j in numbers:
        print(i, j)
Outer loop runs n times
Inner loop runs n times for each outer loop

üëâ O(n¬≤) quadratic time
n = 10 ‚Üí 100 operations

8. Important Clarification

Common Beginner Confusion

This is NOT O(n¬≤):
for num in numbers:
    print(num)

for num in numbers:
    print(num)

This is: n + n = 2n ‚Üí Still O(n)

Pattern Result
Nested loops Multiply
Separate loops Add

9. The Three Most Important Big-O Types

O(1) Constant Access by index
O(n) Linear Single loop
O(n¬≤) Quadratic Nested loops

You'll see these everywhere.

10. How to "Read" Time Complexity

A skill you must build

Look at code and ask:

  1. How many loops?
  2. Are they nested?
  3. What grows with input size?

That's it.

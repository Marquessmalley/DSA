Time Complexity (Big-O)

Zero-Math Version

This is one of the most important lessons in all of DSA.
If this clicks, everything else becomes easier.

1. What Is Time Complexity?

Time complexity is a way to describe how the runtime of an algorithm grows as the input size increases.

It's not about measuring actual seconds ‚Äî it's about understanding the relationship between input size and the number of operations.

Understanding the Notation: O(n)

When you see something like O(n), here's what each part means:

Symbol Meaning
O "Order of" ‚Äî a mathematical notation that describes growth rate
n The input size ‚Äî how many items you're working with

So O(n) reads as "Order of n" and means: the number of operations grows proportionally with the input size.

Common Misconception: O(1) ‚â† "One Operation"

O(1) does not mean "exactly one operation."

It means the number of operations stays constant ‚Äî it doesn't change as input grows. Whether it's 1 operation, 5 operations, or 100 operations, if that number doesn't depend on n, it's O(1). 

def example(arr):
    print(arr[0])      # 1 operation
    print(arr[0] * 2)  # another operation
    print(arr[0] + 10) # another operation
    # Still O(1) ‚Äî always 3 operations, regardless of array size

2. What Problem Does Time Complexity Solve?

Imagine you write a program and ask:

"Will this still work if my input becomes very large?"

Time complexity helps answer:

  • Will it be fast enough?
  • How does runtime grow as input grows?
‚ö†Ô∏è

We are not measuring seconds

We are measuring growth

3. The Core Idea (In One Sentence)

Time complexity describes how the number of operations grows with input size.

Input size is usually called n.

4. Let's Start with the Simplest Example

Example: One Loop
numbers = [2, 4, 6, 8, 10]

for num in numbers:
    print(num)

Let's analyze:

If list size = 5 ‚Üí loop runs 5 times
If list size = 100 ‚Üí loop runs 100 times
If list size = n ‚Üí loop runs n times
üëâ O(n) "order of n"

Meaning:

  • Runtime grows linearly
  • Double the input ‚Üí double the work

5. Why We Ignore Exact Numbers

for num in numbers:
    print(num)      # 1 operation
    print(num * 2)  # another operation

Runs: 2 operations per element = 2n total

We still say: O(n)

Constants (like 2, 5, 100) don't matter for growth

Growth trend matters

6. Constant Time: O(1)

numbers = [10, 20, 30]

print(numbers[0])

No matter:

  • List size = 3
  • List size = 1 million

This always takes the same time.

üëâ O(1) constant time

Arrays (Python Lists)

Your First Data Structure

This is where DSA stops being abstract and starts feeling real.

1. What Is an Array?

An array is a collection of elements stored in contiguous memory locations.

In Python, we use lists, but conceptually they behave like arrays. 

numbers = [10, 20, 30, 40]
Key Characteristics
  • Ordered
  • Indexed (starts at 0)
  • Stores elements of the same conceptual type
  • Fast access by index

2. Why Arrays Exist

Arrays solve a simple problem:

"How do I store many values and access them efficiently?"

Without arrays, you'd need separate variables:

a = 10
b = 20
c = 30
d = 40

Arrays give you:

  • ‚Üí Structure
  • ‚Üí Predictability
  • ‚Üí Performance

3. Indexing: The Superpower of Arrays (O(1))

numbers = [10, 20, 30, 40]

print(numbers[0])  # 10
print(numbers[2])  # 30

No matter how big the array is:

  • Accessing numbers[i] takes the same time
üëâ O(1) index access
üì¨
Mental Model

Imagine a row of mailboxes with numbers on them.
You don't open every mailbox ‚Äî you go directly to the one you want.

Linear Search

Your First Algorithm

This lesson teaches you how algorithms think, not just how code runs.

1. What Is an Algorithm?

An algorithm is:

A step-by-step process to solve a problem.

For this lesson, the problem is simple:

"Given a list and a value, does the value exist?"

2. What Is Linear Search?

Linear search is the simplest searching algorithm. It sequentially checks each element in a collection until it finds the target value or reaches the end of the collection.

It's called "linear" because it progresses through the data in a straight line, one element at a time.

Key Characteristics
  • Works on any list (sorted or unsorted)
  • Requires no preprocessing of the data
  • Simple to understand and implement
  • Guaranteed to find the element if it exists

3. The Problem Setup

You're given:

  • ‚Üí A list (array)
  • ‚Üí A target value
numbers = [4, 9, 15, 23, 42]
target = 15

Your task:

üëâ Check whether target is in numbers

5. The Most Obvious Approach (And That's Okay)

You already know how to do this:

for num in numbers:
    if num == target:
        print("Found")

This approach is an algorithm.

We call it Linear Search.

5. Linear Search ‚Äî Formal Version

def linear_search(arr, target):
    for item in arr:
        if item == target:
            return True
    return False

This function:

  • ‚Üí Returns True if found
  • ‚Üí Returns False otherwise
